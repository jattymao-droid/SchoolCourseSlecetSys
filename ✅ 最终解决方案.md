## 🎯 Nginx 配置错误的原因

宝塔面板提示"检测到被修改的配置文件存在错误"，原因是：

1. **SSL 证书路径不存在**：配置中引用了还未申请的 SSL 证书
2. **include 文件不存在**：引用了宝塔自动生成的 include 文件，但文件可能不存在
3. **配置过于复杂**：包含了太多宝塔特定的占位符和指令

---

## 🔧 解决方案：使用最简 Nginx 配置

### 配置内容（复制使用）

```nginx
server {
    listen 80;
    server_name xk.xmls.vip;
    index index.html;
    root /www/wwwroot/xk.xmls.vip/frontend;
    
    location /prod-api/ {
        proxy_pass http://127.0.0.1:8080/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
    
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    access_log /www/wwwlogs/xk.xmls.vip.log;
    error_log /www/wwwlogs/xk.xmls.vip.error.log;
}
```

### 配置特点

- ✅ **极简设计**：只包含必需的指令
- ✅ **无 SSL**：先测试 HTTP，后续再申请证书
- ✅ **无 include**：不依赖外部文件
- ✅ **无占位符**：所有路径都是实际路径
- ✅ **宝塔兼容**：符合宝塔面板的配置规范

1. **静态文件服务**：`root /www/wwwroot/xk.xmls.vip/frontend;`

## 🐛 常见问题

### Q1: 保存配置后仍然提示错误

**可能原因**：
- 复制时包含了多余的空格或特殊字符
- 使用了 Word 等富文本编辑器复制（会带格式）

**解决方法**：
1. 使用纯文本编辑器（记事本）复制配置
2. 确保配置中没有中文字符（除了注释）
3. 在宝塔面板中点击"重置配置"，然后再粘贴

### Q2: 后端启动后立即停止

**可能原因**：
- 使用了旧的 JAR 包（配置有问题）
- 数据库连接失败
- Redis 连接失败

**解决方法**：
```bash
# 查看完整日志
cat /www/wwwroot/xk.xmls.vip/backend/backend.log

# 检查 PostgreSQL
docker ps | grep postgresql

# 检查 Redis
systemctl status redis

# 测试数据库连接
docker exec postgresql_d6wx-postgresql_D6Wx-1 psql -U postgres -c "SELECT 1"
```

### Q3: 访问网站显示 502 Bad Gateway

**原因**：后端服务未启动或端口不对

**解决方法**：
```bash
# 检查后端是否在运行
curl http://127.0.0.1:8080

# 如果没有响应，查看后端日志
tail -f /www/wwwroot/xk.xmls.vip/backend/backend.log

# 重新启动后端
cd /www/wwwroot/xk.xmls.vip/backend
./restart.sh
```

### Q4: 前端页面显示，但 API 请求失败（404）

**原因**：API 代理配置问题

**解决方法**：
1. 检查 Nginx 配置中的 `location /prod-api/` 是否正确
2. 检查后端是否在 8080 端口运行：`netstat -tlnp | grep 8080`
3. 查看 Nginx 错误日志：`tail -f /www/wwwlogs/xk.xmls.vip.error.log`
