### 3. 问题根源 ⚠️
由于数据库是空的，但导入时报错"学号已存在"，最可能的原因是：

**Excel文件中有重复的学号！**

导入是批量操作，在同一个事务中：
1. 第一次遇到学号 3101 → 验证通过 → 插入成功
2. 第二次遇到学号 3101 → 验证时数据库中已有记录 → 报错"已存在"

## 代码改进

### 改进1：添加批次内重复检查
```java
// 用于检测批次内重复的学号
Set<String> importedStudentNos = new HashSet<>();

// 在验证部分添加
if (importedStudentNos.contains(studentNo)) {
    failureMsg.append("Excel文件中存在重复学号");
    continue;
}

// 验证通过后添加
importedStudentNos.add(studentNo);
```

### 改进2：更详细的错误信息
现在错误信息会明确区分：
- ❌ "Excel文件中存在重复学号" - 批次内重复

- ❌ "用户表中已存在" - 数据库用户表冲突
- ❌ "学生表和用户表中都已存在" - 两个表都冲突
- ❌ "数据库唯一约束冲突" - 数据库层面的约束冲突

## 技术细节

### 导入验证流程
```
1. 检查必填字段（学号、姓名、年级、班级）
   ↓
2. 检查批次内是否重复（新增）
   ↓
3. 检查年级班级是否存在
   ↓
4. 检查数据库中是否已存在
   ├─ 学生表物理检查
   └─ 用户表逻辑检查（del_flag='0'）
   ↓
5. 插入学生和用户记录
   ↓
6. 添加到已导入集合（防止后续重复）
```

### 数据库约束
```sql
-- 学生表唯一约束
"stu_student_info_student_no_key" UNIQUE CONSTRAINT, btree (student_no)
"stu_student_info_user_id_key" UNIQUE CONSTRAINT, btree (user_id)

-- 用户表唯一约束
"sys_user_pkey" PRIMARY KEY, btree (user_id)
```
