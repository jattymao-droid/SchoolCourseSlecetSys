## 部署流程

### 第一步：本地打包


### 第二步：上传到服务器

使用以下任一方式上传：
1. 宝塔面板文件管理
2. SCP 命令: `scp course-selection_*.zip root@your-server-ip:/root/deploy/`
3. FTP 工具（FileZilla、WinSCP）

**目标路径**: `/root/deploy/`

### 第三步：服务器部署

```bash

cd /root/deploy
unzip course-selection_*.zip
cd course-selection_*

# 执行一键部署
chmod +x server-deploy.sh
./server-deploy.sh
```

- **前端**: http://xk.xmls.vip

## Nginx 配置

配置文件: `/www/server/panel/vhost/nginx/xk.xmls.vip.conf`

```nginx
server {
    listen 80;
    server_name xk.xmls.vip;
    
    # 前端静态文件
    location / {
        root /www/wwwroot/course-selection/frontend;
        try_files $uri $uri/ /index.html;
        index index.html;
    }
    
    # 后端 API 代理
    location /prod-api/ {
        proxy_pass http://127.0.0.1:8080/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
    
    client_max_body_size 20M;
}
```

## 性能优化建议

### JVM 参数

根据服务器内存调整（在 start-backend.sh 中）：

```bash
# 2GB 内存服务器
JVM_OPTS="-Xms512m -Xmx1024m -XX:+UseG1GC"

# 4GB 内存服务器
JVM_OPTS="-Xms1g -Xmx2g -XX:+UseG1GC"

# 8GB 内存服务器
JVM_OPTS="-Xms2g -Xmx4g -XX:+UseG1GC"
```

### 数据库连接池


## 故障排查

### 常见问题

1. **后端无法启动**
   - 检查 Java 版本
   - 查看日志文件
   - 检查端口占用

2. **数据库连接失败**
   - 检查 PostgreSQL 状态
   - 验证数据库密码
   - 检查数据库是否存在

3. **页面无法访问**
   - 检查 Nginx 状态
   - 验证域名解析
   - 检查防火墙设置

4. **502 错误**
   - 检查后端是否运行
   - 查看 Nginx 错误日志
   - 检查代理配置
