# 启动项目前的准备工作

## 问题：后端启动失败

后端启动时报错：`Connection refused: localhost/127.0.0.1:6379`

**原因**：Redis 服务未运行

## 解决方案

### 方案 1：启动 Redis（推荐）

#### Windows 系统

1. **如果已安装 Redis**：
   ```powershell
   # 找到 Redis 安装目录，通常在：
   # C:\Program Files\Redis\
   # 或者你自己安装的位置
   
   # 启动 Redis
   redis-server.exe
   ```

2. **如果未安装 Redis**：
   
   下载并安装 Redis for Windows：
   - 下载地址：https://github.com/tporadowski/redis/releases
   - 下载 `Redis-x64-*.zip`
   - 解压到任意目录
   - 运行 `redis-server.exe`

3. **使用 Docker 启动 Redis（推荐）**：
   ```powershell
   docker run -d -p 6379:6379 --name redis redis:latest
   ```

### 方案 2：临时禁用 Redis（不推荐）

如果暂时不需要 Redis 功能，可以修改配置：

1. 编辑 `ruoyi-admin/src/main/resources/application.yml`
2. 注释掉 Redis 配置部分

## 启动步骤

### 1. 启动 Redis

```powershell
# 方式 1：直接运行 Redis
redis-server.exe

# 方式 2：使用 Docker
docker run -d -p 6379:6379 --name redis redis:latest
```

### 2. 验证 Redis 是否运行

```powershell
# 测试连接
redis-cli ping
# 应该返回：PONG
```

### 3. 启动项目

```powershell
# 在 SchoolCourseSlecetSys 目录下运行
.\start-all.ps1
```

## 完整的服务依赖

项目需要以下服务运行：

1. ✅ **PostgreSQL** - 数据库（端口 5432）
2. ❌ **Redis** - 缓存（端口 6379）- 当前未运行
3. ✅ **Java** - 后端运行环境
4. ✅ **Node.js** - 前端开发环境

## 快速检查脚本

创建一个检查脚本来验证所有依赖：

```powershell
# 检查 PostgreSQL
$pg = Get-Process -Name "postgres" -ErrorAction SilentlyContinue
if ($pg) { Write-Host "✓ PostgreSQL 运行中" } else { Write-Host "✗ PostgreSQL 未运行" }

# 检查 Redis
$redis = Get-Process -Name "redis-server" -ErrorAction SilentlyContinue
if ($redis) { Write-Host "✓ Redis 运行中" } else { Write-Host "✗ Redis 未运行" }

# 检查 Java
if (Get-Command java -ErrorAction SilentlyContinue) { Write-Host "✓ Java 已安装" } else { Write-Host "✗ Java 未安装" }

# 检查 Node.js
if (Get-Command node -ErrorAction SilentlyContinue) { Write-Host "✓ Node.js 已安装" } else { Write-Host "✗ Node.js 未安装" }
```
