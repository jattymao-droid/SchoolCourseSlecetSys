# 课程选课系统 - 快速部署说明

## 一、本地打包（Windows）

### 1. 执行打包脚本

```powershell
# 在项目根目录执行
./build-for-production.ps1
```

### 2. 获取部署包

打包完成后，在 `deploy-package` 目录找到：
- 文件名: `course-selection_YYYYMMDD_HHMMSS.zip`
- 大小: 约 50-80 MB

## 二、上传到服务器

### 方法1: 使用宝塔面板

1. 登录宝塔面板
2. 进入"文件"管理
3. 创建目录 `/root/deploy`
4. 上传 zip 文件到该目录

### 方法2: 使用 SCP 命令

```bash
scp course-selection_*.zip root@your-server-ip:/root/deploy/
```

### 方法3: 使用 FTP 工具

- 使用 FileZilla、WinSCP 等工具
- 上传到 `/root/deploy/` 目录

## 三、服务器部署（CentOS 7）

### 1. SSH 登录服务器

```bash
ssh root@your-server-ip
```

### 2. 解压部署包

```bash
cd /root/deploy
unzip course-selection_*.zip
cd course-selection_*
```

### 3. 执行一键部署

```bash
chmod +x server-deploy.sh
./server-deploy.sh
```

### 4. 等待部署完成

脚本会自动完成：
- ✓ 创建目录
- ✓ 部署后端
- ✓ 部署前端
- ✓ 配置数据库
- ✓ 配置 Nginx
- ✓ 启动服务

## 五、常用管理命令

### 服务管理

```bash
# 启动后端
bash /www/wwwroot/course-selection/scripts/start-backend.sh

# 停止后端
bash /www/wwwroot/course-selection/scripts/stop-backend.sh

# 重启后端
bash /www/wwwroot/course-selection/scripts/restart-backend.sh

# 查看状态
bash /www/wwwroot/course-selection/scripts/status.sh
```

### 查看日志

```bash
# 实时查看后端日志
tail -f /www/wwwroot/course-selection/logs/backend.log

# 查看 Nginx 日志
tail -f /www/wwwlogs/xk.xmls.vip.log
```

## 六、配置 HTTPS（可选）

### 使用宝塔面板

1. 登录宝塔面板
2. 进入"网站"
3. 找到 xk.xmls.vip
4. 点击"设置" → "SSL"
5. 申请 Let's Encrypt 免费证书
6. 点击"申请"并等待完成
7. 开启"强制 HTTPS"

## 七、常见问题

### 1. 后端无法启动

```bash
# 检查 Java 版本
java -version

# 查看错误日志
cat /www/wwwroot/course-selection/logs/backend.log
```

### 2. 数据库连接失败

```bash
# 检查 PostgreSQL 状态
systemctl status postgresql

# 启动 PostgreSQL
systemctl start postgresql
```

### 3. 页面无法访问

```bash
# 检查 Nginx 状态
systemctl status nginx

# 重启 Nginx
systemctl restart nginx

# 检查防火墙
firewall-cmd --list-ports
```

### 4. 502 错误

```bash
# 检查后端是否运行
bash /www/wwwroot/course-selection/scripts/status.sh

# 重启后端
bash /www/wwwroot/course-selection/scripts/restart-backend.sh
```

## 八、安全建议

### 1. 修改默认密码

登录系统后立即修改：
- 管理员密码
- 数据库密码

### 2. 配置防火墙

```bash
# 只开放必要端口
firewall-cmd --permanent --add-port=80/tcp
firewall-cmd --permanent --add-port=443/tcp
firewall-cmd --reload
```

### 3. 定期备份

```bash
# 备份数据库
su - postgres -c "pg_dump ry_vue > /backup/ry_vue_$(date +%Y%m%d).sql"
```

## 九、更新部署

### 更新后端

```bash
# 1. 停止服务
bash /www/wwwroot/course-selection/scripts/stop-backend.sh

# 2. 备份旧版本
cp /www/wwwroot/course-selection/backend/ruoyi-admin.jar \
   /www/wwwroot/course-selection/backend/ruoyi-admin.jar.bak

# 3. 上传新的 JAR 文件（使用 FTP 或 SCP）

# 4. 启动服务
bash /www/wwwroot/course-selection/scripts/start-backend.sh
```

### 更新前端

```bash
# 1. 备份旧版本
cp -r /www/wwwroot/course-selection/frontend \
      /www/wwwroot/course-selection/frontend.bak

# 2. 上传新的前端文件（使用 FTP 或 SCP）

# 3. 清除浏览器缓存（Ctrl + F5）
```

## 十、技术支持

### 服务器环境要求

- CentOS 7
- PostgreSQL (端口 5432)
- Redis (端口 6379)
- Nginx
- Java 1.8+

### 目录结构

```
/www/wwwroot/course-selection/
├── backend/          # 后端应用
├── frontend/         # 前端文件
├── logs/            # 日志目录
├── uploadPath/      # 上传文件
└── scripts/         # 管理脚本
```

### 配置文件位置

- Nginx: `/www/server/panel/vhost/nginx/xk.xmls.vip.conf`
- 后端配置: 内置在 JAR 文件中
- 数据库: PostgreSQL 默认配置
