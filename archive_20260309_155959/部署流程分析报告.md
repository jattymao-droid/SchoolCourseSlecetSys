## ⚠️ 发现的潜在问题

### 问题 1: Redis 连接配置未明确
**严重程度**: 🔴 高

**问题描述**:
- 部署脚本检测到 Redis Docker 容器
- 但后端配置文件中没有明确 Redis 的连接配置
- 如果 Redis 容器使用了非默认端口或需要密码，后端可能无法连接

**影响**:
- 后端启动可能失败
- 缓存功能无法使用
- 会话管理可能出问题

**建议解决方案**:
需要检查后端的 Redis 配置文件，确保：
1. Redis 主机地址正确（Docker 容器的话可能需要容器名或 IP）
2. Redis 端口正确（默认 6379）
3. Redis 密码配置（如果容器设置了密码）

**需要检查的文件**:
- `ruoyi-admin/src/main/resources/application-prod.yml`
- `ruoyi-admin/src/main/resources/application.yml`

---

### 问题 2: Docker 容器网络连接
**严重程度**: 🟡 中

**问题描述**:
- 后端应用运行在宿主机上（非容器）
- PostgreSQL 和 Redis 运行在 Docker 容器中
- 后端配置使用 `localhost` 连接数据库

**潜在风险**:
- 如果 Docker 容器没有映射端口到宿主机，连接会失败
- 需要确保容器端口映射：
  - PostgreSQL: `-p 5432:5432`
  - Redis: `-p 6379:6379`

**验证命令**:
```bash
# 检查容器端口映射
docker ps --format "table {{.Names}}\t{{.Ports}}"

# 应该看到类似输出：
# postgres    0.0.0.0:5432->5432/tcp
# redis       0.0.0.0:6379->6379/tcp
```

**建议**:
在部署脚本中添加端口映射检查

---

### 问题 3: 前端 HTTPS 配置但 Nginx 只配置了 HTTP
**严重程度**: 🟡 中

**问题描述**:
- 前端配置的 API 地址: `https://xk.xmls.vip/prod-api`
- Nginx 配置只监听 80 端口（HTTP）
- 没有配置 SSL 证书

**影响**:
- 如果用户通过 HTTPS 访问，API 请求会失败（混合内容错误）
- 浏览器会阻止 HTTPS 页面调用 HTTP API

**解决方案**:
1. 配置 SSL 证书（Let's Encrypt）
2. 修改 Nginx 配置添加 443 端口监听
3. 或者修改前端配置改为 HTTP（不推荐）

**建议操作**:
```bash
# 在宝塔面板中配置 SSL
# 网站 → xk.xmls.vip → SSL → Let's Encrypt → 申请
```

### 问题 4: 数据库导入错误处理不够详细
**严重程度**: 🟢 低

**问题描述**:
- 数据库导入失败时，错误输出被重定向到 `/dev/null`
- 用户无法看到具体的错误信息

**当前代码**:
```bash
docker exec -e PGPASSWORD="${DB_PASSWORD}" "${POSTGRES_CONTAINER}" psql -U ${DB_USER} -d ${DB_NAME} -f /tmp/database_backup.sql > /dev/null 2>&1
```

**建议改进**:
```bash
# 保存错误日志
docker exec -e PGPASSWORD="${DB_PASSWORD}" "${POSTGRES_CONTAINER}" psql -U ${DB_USER} -d ${DB_NAME} -f /tmp/database_backup.sql 2>&1 | tee /tmp/db_import.log
if [ ${PIPESTATUS[0]} -ne 0 ]; then
    echo -e "${RED}错误: 数据库导入失败${NC}"
    echo "错误日志已保存到: /tmp/db_import.log"
    tail -20 /tmp/db_import.log
    exit 1
fi
```

### 问题 5: 文件上传目录权限
**严重程度**: 🟢 低

**问题描述**:
- 部署脚本创建了 `uploadPath` 目录
- 但没有设置正确的权限
- Java 应用可能没有写入权限

**建议添加**:
```bash

chown -R root:root "${APP_DIR}"
chmod -R 755 "${APP_DIR}"
chmod -R 777 "${UPLOAD_DIR}"  # 上传目录需要写权限
```

### 问题 6: 后端启动时没有等待数据库就绪
**严重程度**: 🟡 中

**问题描述**:
- 部署脚本导入数据库后立即启动后端
- 如果数据库还在处理导入，后端可能连接失败

**建议改进**:
```bash
# 在启动后端前添加数据库连接测试
echo "等待数据库就绪..."
for i in {1..30}; do
    if docker exec -e PGPASSWORD="${DB_PASSWORD}" "${POSTGRES_CONTAINER}" psql -U ${DB_USER} -d ${DB_NAME} -c "SELECT 1" > /dev/null 2>&1; then
        echo "数据库连接成功"
        break
    fi
    if [ $i -eq 30 ]; then
        echo -e "${RED}错误: 数据库连接超时${NC}"
        exit 1
    fi
    sleep 1
done
```
