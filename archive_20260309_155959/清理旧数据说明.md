# 清理旧的已删除用户数据

## 问题说明

虽然我们已经修复了代码，让删除学生时物理删除用户记录，但**数据库中可能还有之前逻辑删除的用户记录**（`del_flag = '2'`）。

这些旧记录会导致：
- 无法重新导入相同学号的学生
- 报错："学号 XXX 已存在（包含物理冲突或账号冲突）"

## 解决方案

需要清理数据库中已逻辑删除的用户记录。

### 方式 1：使用 PowerShell 脚本（推荐）

```powershell
# 在 SchoolCourseSlecetSys 目录下运行
.\clean-deleted-users.ps1
```

脚本会：
1. 检查有多少已删除的学生用户
2. 显示这些用户的信息
3. 询问是否删除
4. 物理删除这些记录
5. 验证清理结果

### 方式 2：手动执行 SQL

如果 PowerShell 脚本无法运行，可以手动执行 SQL：

1. 连接数据库：
```bash
# 本地开发环境
psql -h localhost -p 5432 -U postgres -d ry_vue

# 生产环境
psql -h localhost -p 5432 -U postgres -d school-course
```

2. 查看有多少已删除的用户：
```sql
SELECT COUNT(*) FROM sys_user WHERE del_flag = '2' AND user_type = '01';
```

3. 查看这些用户的详细信息：
```sql
SELECT user_id, user_name, nick_name FROM sys_user 
WHERE del_flag = '2' AND user_type = '01' 
LIMIT 10;
```

4. 删除用户角色关联：
```sql
DELETE FROM sys_user_role WHERE user_id IN (
    SELECT user_id FROM sys_user WHERE del_flag = '2' AND user_type = '01'
);
```

5. 删除用户岗位关联：
```sql
DELETE FROM sys_user_post WHERE user_id IN (
    SELECT user_id FROM sys_user WHERE del_flag = '2' AND user_type = '01'
);
```

6. 删除用户记录：
```sql
DELETE FROM sys_user WHERE del_flag = '2' AND user_type = '01';
```

7. 验证清理结果：
```sql
SELECT COUNT(*) FROM sys_user WHERE del_flag = '2' AND user_type = '01';
-- 应该返回 0
```

## 清理后的效果

✅ 数据库中不再有已删除的学生用户记录
✅ 可以重新导入任何学号的学生
✅ 不会再报"学号已存在"的错误

## 注意事项

1. **备份数据**：清理前建议备份数据库
2. **只删除学生**：脚本只删除 `user_type = '01'`（学生）的已删除用户
3. **不影响其他用户**：教师、管理员等其他用户不受影响
4. **永久删除**：物理删除后无法恢复

## 为什么会有这个问题？

1. 之前的代码使用逻辑删除（软删除）
2. 删除学生时，用户记录只是标记为已删除（`del_flag = '2'`）
3. 记录仍然存在数据库中
4. 重新导入时检查到这些记录，导致冲突

## 修复后的行为

从现在开始：
- 删除学生时，用户记录会被**物理删除**
- 不会再产生已删除的用户记录
- 可以随时重新导入相同学号的学生

## 故障排查

如果清理后仍然无法导入：

1. 检查是否还有已删除的记录：
```sql
SELECT * FROM sys_user WHERE del_flag = '2' AND user_type = '01';
```

2. 检查学生表中是否有重复学号：
```sql
SELECT student_no, COUNT(*) 
FROM stu_student_info 
GROUP BY student_no 
HAVING COUNT(*) > 1;
```

3. 重启后端服务，确保使用最新代码
