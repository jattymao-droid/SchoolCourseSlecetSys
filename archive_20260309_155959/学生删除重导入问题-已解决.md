# 学生删除重导入问题 - 已解决 ✅

## 问题描述

删除学生后重新导入报错：
```
导入失败！共 290 条数据格式不正确：
学号 3101 已存在（包含物理冲突或账号冲突）
学号 3102 已存在（包含物理冲突或账号冲突）
...
```

## 根本原因

1. **旧代码使用逻辑删除**：
   - 删除学生时，`stu_student_info` 表只标记 `del_flag = '2'`
   - 删除用户时，`sys_user` 表也只标记 `del_flag = '2'`
   - 记录仍在数据库中
2. **导入检查冲突**：重新导入时检测到这些已删除的记录，导致冲突

## 解决方案

### 1. 代码修复（已完成）

修改了以下文件，将逻辑删除改为物理删除：

**学生表删除：**
- `StuStudentInfoMapper.xml` - 将 `deleteStudentById` 从 UPDATE 改为 DELETE

**用户表删除：**
- `StudentServiceImpl.java` - 修改删除逻辑，调用物理删除方法
- `SysUserMapper.java` - 添加物理删除方法
- `SysUserMapper.xml` - 添加物理删除 SQL

### 2. 数据库清理（已完成）

清理了数据库中的旧记录：

```sql
-- 清理已逻辑删除的学生记录
DELETE FROM stu_student_info WHERE del_flag = '2';  -- 583 条

-- 清理已逻辑删除的用户记录
DELETE FROM sys_user_role WHERE user_id IN (...);  -- 0 条
DELETE FROM sys_user_post WHERE user_id IN (...);  -- 0 条
DELETE FROM sys_user WHERE del_flag = '2' AND user_type = '01';  -- 577 条
```

验证结果：
```sql
-- 学生表
SELECT COUNT(*) FROM stu_student_info WHERE del_flag = '2';
-- 返回 0 ✅

-- 用户表
SELECT COUNT(*) FROM sys_user WHERE del_flag = '2' AND user_type = '01';
-- 返回 0 ✅
```
