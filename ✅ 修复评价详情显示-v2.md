# ✅ 修复评价详情显示 - v2

## 问题描述

教师考核列表中点击"评价详情"按钮后，弹窗中没有显示任何内容（空白）。

---

## 问题原因

在 `selectTeacherEvaluationDetail` 查询中存在两个问题：

### 问题 1：使用 LEFT JOIN 导致条件失效

```xml
<!-- 错误的查询 -->
FROM stu_evaluation e
LEFT JOIN cou_course c ON e.course_id = c.id  -- ❌ LEFT JOIN
WHERE c.semester_id = #{semesterId}           -- ❌ 当 c 为 NULL 时条件失败
```

当使用 LEFT JOIN 时，如果没有匹配的课程记录，`c` 的所有字段都是 NULL，导致 `WHERE c.semester_id = #{semesterId}` 条件失败，查询不到任何结果。

### 问题 2：WHERE 条件使用了 JOIN 表的字段

```xml
WHERE c.semester_id = #{semesterId}  -- ❌ 应该使用 e.semester_id
```

应该直接从 `stu_evaluation` 表（主表）过滤，而不是从 JOIN 的表过滤。

---

## 修复方案

### 修复后的查询

```xml
<select id="selectTeacherEvaluationDetail" resultMap="TeacherEvaluationDetailResult">
    SELECT
        e.student_id,
        stu.real_name AS student_name,
        stu.student_no,
        cla.class_name,
        e.course_id,
        c.course_name,
        e.question_id,
        q.content AS question_content,
        e.score,
        q.score AS max_score,
        e.create_time
    FROM stu_evaluation e
    -- ✅ 改为 INNER JOIN，确保有对应的课程
    INNER JOIN cou_course c ON e.course_id = c.id AND c.del_flag = '0'
    LEFT JOIN eva_question q ON e.question_id = q.id
    LEFT JOIN stu_student_info stu ON e.student_id = stu.user_id AND stu.del_flag = '0'
    LEFT JOIN sys_class cla ON stu.class_id = cla.id
    -- ✅ 直接从评价表过滤
    WHERE e.semester_id = #{semesterId}
    <if test="teacherId != null">
        AND c.teacher_id = #{teacherId}
    </if>
    <if test="teacherName != null and teacherName != ''">
        AND c.teacher_name = #{teacherName}
    </if>
    ORDER BY e.create_time DESC
</select>
```

### 修复要点

1. ✅ **INNER JOIN cou_course**：确保只查询有对应课程的评价记录
2. ✅ **WHERE e.semester_id**：直接从评价表过滤，避免 NULL 值问题
3. ✅ **c.del_flag = '0'**：确保课程未被删除
4. ✅ **保留其他 LEFT JOIN**：学生信息、班级信息、问题信息可能不存在，使用 LEFT JOIN
