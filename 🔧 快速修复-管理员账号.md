# 🔧 快速修复 - 管理员账号

## 问题

登录时提示：**用户名或密码错误，请检查导入的数据库文件是否完整**

## 原因

数据库中缺少 `admin` 管理员账号（user_id=1）

---

## 🚀 快速修复（3 种方法任选其一）

### 方法 1：一键修复脚本（最简单）

```bash
# 在服务器上执行
cd /www/wwwroot/xk.xmls.vip/scripts
chmod +x fix-admin.sh
./fix-admin.sh
```

**预期输出**：
```
✅ 修复完成！
现在可以使用以下账号登录：
  用户名：admin
  密码：admin123
```

---

### 方法 2：直接执行 SQL（推荐）

```bash
# 在服务器上执行（复制整段命令）
docker exec -i postgresql_d6wx-postgresql_D6Wx-1 psql -U postgres -d school-course << 'EOF'
DELETE FROM public.sys_user WHERE user_name = 'admin';
INSERT INTO public.sys_user (user_id, dept_id, user_name, nick_name, user_type, email, phonenumber, sex, avatar, password, status, del_flag, login_ip, login_date, pwd_update_date, create_by, create_time, update_by, update_time, remark) VALUES (1, 103, 'admin', '管理员', '00', 'admin@ruoyi.com', '15888888888', '1', '', '$2a$10$7JB720yubVSZvUI0rEqK/.VqGOZTH.ulu33dHOiBE8ByOhJIrdAu2', '0', '0', '127.0.0.1', NOW(), NOW(), 'admin', NOW(), '', NULL, '管理员');
DELETE FROM public.sys_user_role WHERE user_id = 1;
INSERT INTO public.sys_user_role (user_id, role_id) VALUES (1, 1);
SELECT setval('sys_user_user_id_seq', (SELECT MAX(user_id) FROM sys_user));
SELECT user_id, user_name, nick_name, status FROM public.sys_user WHERE user_name = 'admin';
EOF
```

**预期输出**：
```
 user_id | user_name | nick_name | status
---------+-----------+-----------+--------
       1 | admin     | 管理员    | 0
(1 row)
```

## 🐛 如果还是无法登录

### 检查 1：用户状态

```bash
docker exec postgresql_d6wx-postgresql_D6Wx-1 psql -U postgres -d school-course -c "SELECT user_id, user_name, status, del_flag FROM sys_user WHERE user_name = 'admin';"
```

- `status` 应该是 `0`（正常）
- `del_flag` 应该是 `0`（未删除）

如果不是，执行：
```bash
docker exec postgresql_d6wx-postgresql_D6Wx-1 psql -U postgres -d school-course -c "UPDATE sys_user SET status = '0', del_flag = '0' WHERE user_name = 'admin';"
```

### 检查 2：角色分配

```bash
docker exec postgresql_d6wx-postgresql_D6Wx-1 psql -U postgres -d school-course -c "SELECT * FROM sys_user_role WHERE user_id = 1;"
```

应该有记录：`user_id=1, role_id=1`

如果没有，执行：
```bash
docker exec postgresql_d6wx-postgresql_D6Wx-1 psql -U postgres -d school-course -c "INSERT INTO sys_user_role (user_id, role_id) VALUES (1, 1);"
```

### 检查 3：后端日志

```bash
tail -50 /www/wwwroot/xk.xmls.vip/backend/backend.log
```

查看是否有数据库连接错误或其他异常。

### 检查 4：浏览器控制台

按 F12 打开浏览器开发者工具，查看 Console 和 Network 标签，看是否有 API 请求错误。
