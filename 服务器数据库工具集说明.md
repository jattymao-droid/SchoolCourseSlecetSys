# 服务器数据库工具集完整说明

## 工具概览

我为你创建了一套完整的PostgreSQL数据库管理工具，包括诊断、修复、管理和清理功能。

## 工具列表

### 1. 🔍 诊断工具

#### diagnose-database-server.sh
**用途**: 全面诊断数据库状态，找出配置问题

**功能**:
- 列出所有数据库及大小
- 统计每个数据库的表数量
- 查看当前数据库连接
- 检查后端应用进程
- 分析配置文件
- 自动给出修复建议

**使用方法**:
```bash
chmod +x diagnose-database-server.sh
./diagnose-database-server.sh
```

**适用场景**:
- 不确定实际使用哪个数据库
- 数据库管理工具连接失败
- 后端应用无法连接数据库
- 需要全面了解数据库状态

---

### 2. 🔧 自动修复工具

#### auto-fix-database-server.sh
**用途**: 自动检测并修复数据库配置问题

**功能**:
- 自动检测数据库状态
- 识别配置问题类型
- 提供多种修复方案
- 一键执行修复操作

**使用方法**:
```bash
chmod +x auto-fix-database-server.sh
./auto-fix-database-server.sh
```

**修复场景**:
1. **数据库名称不匹配**: 自动修改管理脚本配置
2. **空数据库**: 自动从备份文件导入数据
3. **数据库复制**: 从 ry_vue 复制到 school-course

**适用场景**:
- 快速修复数据库配置问题
- 不想手动执行复杂命令
- 需要批量修改配置文件

---

### 3. 📊 数据库管理工具

#### manage-database-server.sh
**用途**: 交互式数据库管理工具，功能最全面

**主要功能**:
1. 查看数据库信息（大小、表数量、连接数、版本）
2. 列出所有表
3. 管理指定表
   - 查看表结构
   - 查看表统计
   - 查看表数据
   - 清空表数据
   - 删除表
   - 备份表
4. 清除逻辑删除记录
5. 执行自定义SQL
6. 备份整个数据库
7. 查看数据库连接
8. 查看表空间使用情况
9. 优化数据库（VACUUM）

**使用方法**:
```bash
chmod +x manage-database-server.sh
./manage-database-server.sh
```

**特色功能**:
- 彩色输出，界面友好
- 交互式菜单，易于操作
- 自动检测PostgreSQL容器
- 支持所有常用数据库操作

**适用场景**:
- 日常数据库维护
- 查看和分析表数据
- 执行数据库操作
- 数据库性能优化

---

### 4. 🧹 数据清理工具

#### clean-database-server.sh
**用途**: 通用数据库清理工具，支持多种清理模式

**清理模式**:
1. 仅清除逻辑删除记录（del_flag='2'）
2. 清除所有学生数据
3. 清除所有课程数据
4. 清除所有选课数据
5. 清除所有业务数据（学生+课程+选课）

**使用方法**:
```bash
chmod +x clean-database-server.sh
./clean-database-server.sh
```

**安全特性**:
- 所有操作需要确认
- 显示清理前的统计信息
- 按外键依赖顺序删除

**适用场景**:
- 清理测试数据
- 重置数据库
- 清除特定类型的数据

---

#### clean-deleted-records-server.sh
**用途**: 专门清除逻辑删除记录

**功能**:
- 清除 stu_student_info 表中 del_flag='2' 的记录
- 清除 sys_user 表中 del_flag='2' 的记录
- 统计和验证清除结果

**使用方法**:
```bash
chmod +x clean-deleted-records-server.sh
./clean-deleted-records-server.sh
```

**适用场景**:
- 删除学生后重新导入报错
- 清理逻辑删除的记录
- 释放数据库空间

## 快速开始指南

### 场景 1: 首次使用，不确定数据库状态

```bash
# 1. 上传所有脚本到服务器
scp *.sh root@your-server:/root/

# 2. 添加执行权限
chmod +x *.sh

# 3. 运行诊断
./diagnose-database-server.sh

# 4. 根据诊断结果，运行自动修复
./auto-fix-database-server.sh

# 5. 使用管理工具
./manage-database-server.sh
```

### 场景 2: 数据库管理工具显示 0 rows

```bash
# 直接运行自动修复工具
./auto-fix-database-server.sh

# 选择合适的修复方案
# 方案1: 修改脚本连接到正确的数据库
# 方案2: 重新导入数据
```

### 场景 3: 清除逻辑删除记录

```bash
# 方法1: 使用专用清理工具
./clean-deleted-records-server.sh

# 方法2: 使用管理工具
./manage-database-server.sh
# 选择 4 - 清除逻辑删除记录
```

### 场景 4: 日常数据库维护

```bash
./manage-database-server.sh

# 常用操作：
# - 选择 1: 查看数据库信息
# - 选择 2: 列出所有表
# - 选择 8: 查看表空间使用情况
# - 选择 9: 优化数据库
```

- 所有删除操作都需要输入 `yes` 确认
