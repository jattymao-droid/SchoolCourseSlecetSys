# ✅ 修复教师考核评分显示问题

## 问题描述

学生提交了课程评价，但是：
1. ❌ 教师考核页面不显示评价得分（显示为 0 或空）
2. ❌ 评价详情页面没有内容

## 问题原因

在 `TeacherAssessmentMapper.xml` 的 `selectTeacherAssessmentList` 查询中：
- `avg_score` 硬编码为 `0.0`
- `eval_count` 硬编码为 `0`
- 没有关联 `stu_evaluation` 表来计算实际的评价数据

```xml
<!-- 错误的查询 -->
SELECT
    c.teacher_id,
    COALESCE(c.teacher_name, u.nick_name, '未知教师') AS teacher_name,
    s.id AS semester_id,
    s.semester_name,
    0.0 AS avg_score,          -- ❌ 硬编码为 0
    0 AS eval_count,            -- ❌ 硬编码为 0
    COUNT(DISTINCT c.id) AS course_count
FROM cou_course c
INNER JOIN sys_semester s ON c.semester_id = s.id
LEFT JOIN sys_user u ON c.teacher_id = u.user_id
-- ❌ 没有关联 stu_evaluation 表
WHERE c.del_flag = '0'
  AND c.teacher_id IS NOT NULL
GROUP BY c.teacher_id, c.teacher_name, u.nick_name, s.id, s.semester_name
```

## 修复方案

### 1. 修复 SQL 查询

在 `TeacherAssessmentMapper.xml` 中修复 `selectTeacherAssessmentList` 查询：

```xml
<!-- 修复后的查询 -->
<select id="selectTeacherAssessmentList" parameterType="Long" resultMap="TeacherAssessmentResult">
    SELECT
        c.teacher_id,
        COALESCE(c.teacher_name, u.nick_name, '未知教师') AS teacher_name,
        s.id AS semester_id,
        s.semester_name,
        -- ✅ 计算平均得分：(学生得分/题目满分)*10，保留1位小数
        COALESCE(ROUND(AVG(e.score * 1.0 / NULLIF(q.score, 0) * 10), 1), 0.0) AS avg_score,
        -- ✅ 统计评价人次
        COUNT(DISTINCT e.id) AS eval_count,
        COUNT(DISTINCT c.id) AS course_count
    FROM cou_course c
    INNER JOIN sys_semester s ON c.semester_id = s.id
    LEFT JOIN sys_user u ON c.teacher_id = u.user_id
    -- ✅ 关联评价表
    LEFT JOIN stu_evaluation e ON e.course_id = c.id AND e.semester_id = c.semester_id
    -- ✅ 关联问题表（获取满分）
    LEFT JOIN eva_question q ON e.question_id = q.id
    WHERE c.del_flag = '0'
      AND c.teacher_id IS NOT NULL
    <if test="semesterId != null">
        AND c.semester_id = #{semesterId}
    </if>
    GROUP BY c.teacher_id, c.teacher_name, u.nick_name, s.id, s.semester_name
    -- ✅ 按平均得分降序排序
    ORDER BY avg_score DESC NULLS LAST, c.teacher_name, s.semester_name
</select>
```

### 2. 评分计算逻辑

**评分公式**：
```
平均得分 = AVG(学生得分 / 题目满分 * 10)
```

**示例**：
- 题目满分：5 分
- 学生 A 评分：4 分 → 换算为 10 分制：4/5*10 = 8.0 分
- 学生 B 评分：5 分 → 换算为 10 分制：5/5*10 = 10.0 分
- 平均得分：(8.0 + 10.0) / 2 = 9.0 分

**处理边界情况**：
- 使用 `NULLIF(q.score, 0)` 避免除以 0
- 使用 `COALESCE(..., 0.0)` 处理 NULL 值
- 使用 `ROUND(..., 1)` 保留 1 位小数

## 技术说明

### 为什么要换算为 10 分制？

评价题目的满分可能不同（如 5 分、10 分等），为了统一比较，需要换算为 10 分制：
- 5 分制题目：得分 4 分 → 换算为 8.0 分（4/5*10）
- 10 分制题目：得分 8 分 → 换算为 8.0 分（8/10*10）

### PostgreSQL 特定语法

- `NULLIF(q.score, 0)`：如果 q.score 为 0，返回 NULL，避免除以 0
- `COALESCE(..., 0.0)`：如果结果为 NULL，返回 0.0
- `ROUND(..., 1)`：保留 1 位小数
- `NULLS LAST`：NULL 值排在最后

### 性能优化

- 使用 `LEFT JOIN` 而不是子查询
- 使用 `COUNT(DISTINCT e.id)` 避免重复计数
- 在 `WHERE` 子句中过滤，而不是在 `HAVING` 中

## 常见问题

### Q1: 修复后仍然显示 0

**检查**：
1. 确认数据库中有评价数据：
   ```sql
   SELECT COUNT(*) FROM stu_evaluation WHERE semester_id = 学期ID;
   ```
2. 确认 JAR 包已更新（查看文件时间戳）
3. 确认后端已重启（查看日志中的启动时间）

### Q2: 评价详情页面仍然为空

**检查**：
1. 评价详情查询是否正确（`selectTeacherEvaluationDetail`）
2. 数据库中 `stu_evaluation` 表是否有数据
3. 浏览器控制台是否有 API 错误

### Q3: 平均得分计算不准确

**检查**：
1. 评价题目的满分设置是否正确（`eva_question.score`）
2. 学生评分是否在合理范围内
3. 是否有除以 0 的情况
