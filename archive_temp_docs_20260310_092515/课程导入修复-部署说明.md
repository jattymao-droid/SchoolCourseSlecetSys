# 课程导入事务错误修复 - 部署说明

## 问题描述

在云服务器上导入课程时报错：
```
ERROR: current transaction is aborted, commands ignored until end of transaction block
```

## 修复内容

已修复 `CourseServiceImpl.java` 中的事务处理问题：

### 修改的方法

1. **findGradeByName** - 添加 `@Transactional(propagation = Propagation.NOT_SUPPORTED, readOnly = true)`
2. **findSemesterByName** - 添加 `@Transactional(propagation = Propagation.NOT_SUPPORTED, readOnly = true)`
3. **findClassByGradeAndShortName** - 添加 `@Transactional(propagation = Propagation.NOT_SUPPORTED, readOnly = true)`
4. **findUserByNickName** - 添加 `@Transactional(propagation = Propagation.NOT_SUPPORTED, readOnly = true)`
5. **findOrCreateTeacherUser** - 拆分为两个方法：
   - `findTeacherUser` - 只读查询（NOT_SUPPORTED）
   - `createTeacherUser` - 创建用户（REQUIRES_NEW）

### 修复原理

- 将查询方法设置为非事务执行（`Propagation.NOT_SUPPORTED`）
- 避免在主事务中断后，查询SQL被PostgreSQL拒绝执行
- 创建用户的操作在新事务中执行（`Propagation.REQUIRES_NEW`）

## 技术说明

### 事务传播行为

- **NOT_SUPPORTED**: 以非事务方式执行，如果当前存在事务，则挂起当前事务
- **REQUIRES_NEW**: 创建一个新事务，如果当前存在事务，则挂起当前事务

### 为什么这样修复有效？

1. **查询方法不参与事务**: 将查询方法设置为 `NOT_SUPPORTED`，使其在事务外执行
2. **避免事务中断**: 即使主事务中有错误，查询方法仍然可以正常执行
3. **创建操作独立**: 创建用户的操作在新事务中执行，不影响主事务
