# 快速导入数据库命令

## 问题

导入脚本报错：`ERROR: database "school-course" already exists`

**原因**：数据库已存在，但删除失败（可能有活动连接）

---

## 🚀 快速解决（复制执行）

### 方法 1：强制删除并导入（推荐）

```bash
# 1. 先停止后端服务（断开数据库连接）
cd /www/wwwroot/xk.xmls.vip/backend
./stop.sh

# 2. 强制断开所有数据库连接
docker exec postgresql_d6wx-postgresql_D6Wx-1 psql -U postgres -c "SELECT pg_terminate_backend(pid) FROM pg_stat_activity WHERE datname = 'school-course' AND pid <> pg_backend_pid();"

# 3. 删除数据库
docker exec postgresql_d6wx-postgresql_D6Wx-1 psql -U postgres -c "DROP DATABASE IF EXISTS \"school-course\";"

# 4. 创建新数据库
docker exec postgresql_d6wx-postgresql_D6Wx-1 psql -U postgres -c "CREATE DATABASE \"school-course\" WITH ENCODING='UTF8';"

# 5. 导入数据
docker exec -i postgresql_d6wx-postgresql_D6Wx-1 psql -U postgres -d school-course < /www/wwwroot/xk.xmls.vip/sql/complete-database.sql

# 6. 验证导入
docker exec postgresql_d6wx-postgresql_D6Wx-1 psql -U postgres -d school-course -c "SELECT COUNT(*) FROM information_schema.tables WHERE table_schema = 'public';"

# 7. 启动后端
cd /www/wwwroot/xk.xmls.vip/backend
./start.sh
tail -f backend.log
```

---

### 方法 2：直接清空并导入（不删除数据库）

```bash
# 1. 停止后端服务
cd /www/wwwroot/xk.xmls.vip/backend

# 2. 删除所有表（清空数据库）
```

## 🐛 如果还有问题

### 问题 1：仍然无法删除数据库

**解决**：重启 PostgreSQL 容器
```bash
docker restart postgresql_d6wx-postgresql_D6Wx-1
# 等待 10 秒
sleep 10
# 然后重新执行导入命令
```

**解决**：确保使用 postgres 超级用户
