# 服务器数据库清理脚本使用说明

## 脚本列表

### 1. clean-deleted-records-server.sh
**用途**: 清除逻辑删除记录（del_flag='2'）

**适用场景**:
- 删除学生后重新导入时报"学号已存在"错误
- 清理数据库中的逻辑删除记录

**清理内容**:
- `stu_student_info` 表中 `del_flag='2'` 的记录
- `sys_user` 表中 `del_flag='2'` 的记录

### 2. clean-database-server.sh
**用途**: 通用数据库清理工具（多种清理模式）

**清理模式**:
1. 仅清除逻辑删除记录 (del_flag='2')
2. 清除所有学生数据
3. 清除所有课程数据
4. 清除所有选课数据
5. 清除所有业务数据 (学生+课程+选课)

## 使用方法

### 步骤 1: 上传脚本到服务器

```bash
# 使用 scp 上传脚本
scp clean-deleted-records-server.sh root@your-server:/root/
scp clean-database-server.sh root@your-server:/root/
```

或者使用宝塔面板的文件管理功能上传。

### 步骤 2: 添加执行权限

```bash
# SSH 登录服务器后执行
chmod +x clean-deleted-records-server.sh
chmod +x clean-database-server.sh
```

### 步骤 3: 运行脚本

#### 方式 1: 清除逻辑删除记录（推荐）

```bash
./clean-deleted-records-server.sh
```

脚本会自动：
1. 检测 PostgreSQL Docker 容器
2. 统计逻辑删除记录数量
3. 请求确认
4. 清除记录
5. 验证清除结果

#### 方式 2: 使用通用清理工具

```bash
./clean-database-server.sh
```

根据提示选择清理模式。

## 脚本特点

### 自动检测容器
脚本会自动检测运行中的 PostgreSQL Docker 容器，无需手动指定容器名称。

### 安全确认
所有清理操作都需要用户确认，防止误操作。

### 详细日志
脚本会显示详细的操作日志，包括：
- 检测到的容器名称
- 数据库连接状态
- 清理前的记录统计
- 清理后的验证结果

### 错误处理
脚本使用 `set -e` 确保遇到错误时立即退出，防止数据损坏。

## 配置说明

脚本中的默认配置：

```bash
DB_NAME="school-course"      # 数据库名称
DB_USER="postgres"           # 数据库用户
DB_PASSWORD="mm5621528"      # 数据库密码
```

如果你的配置不同，请修改脚本中的这些变量。

## 常见问题

### Q1: 提示"未找到运行中的PostgreSQL容器"

**解决方法**:
```bash
# 检查容器是否运行
docker ps | grep postgres

# 如果容器未运行，启动容器
docker start <container_name>
```

### Q2: 提示"数据库连接失败"

**可能原因**:
1. 数据库名称错误（检查是否为 `school-course`）
2. 密码错误（检查是否为 `mm5621528`）
3. 容器内数据库未启动

**解决方法**:
```bash
# 进入容器检查
docker exec -it <container_name> psql -U postgres -l
```

### Q3: 清除后仍然报"学号已存在"

**可能原因**:
1. 后端服务缓存了旧数据
2. 数据库中仍有物理记录

**解决方法**:
```bash
# 1. 重启后端服务
systemctl restart course-backend

# 2. 手动检查数据库
docker exec -it <container_name> psql -U postgres -d school-course
# 在 psql 中执行：
SELECT student_no, del_flag FROM stu_student_info WHERE student_no = '3101';
```

## 注意事项

1. **备份数据**: 在执行清理操作前，建议先备份数据库
2. **停止服务**: 建议在清理数据时停止后端服务，避免数据不一致
3. **确认操作**: 所有清理操作都需要输入 `yes` 确认，输入其他内容会取消操作
4. **权限要求**: 脚本需要有执行权限（`chmod +x`）
5. **Docker 环境**: 脚本依赖 Docker 命令，确保 Docker 已安装并运行

## 相关命令

### 手动清除逻辑删除记录

```bash
# 进入 PostgreSQL 容器
docker exec -it <container_name> psql -U postgres -d school-course

# 执行 SQL
DELETE FROM stu_student_info WHERE del_flag = '2';
DELETE FROM sys_user WHERE del_flag = '2';
```

### 查看逻辑删除记录

```bash
# 查看学生表
docker exec -it <container_name> psql -U postgres -d school-course -c \
  "SELECT COUNT(*) FROM stu_student_info WHERE del_flag = '2';"

# 查看用户表
docker exec -it <container_name> psql -U postgres -d school-course -c \
  "SELECT COUNT(*) FROM sys_user WHERE del_flag = '2';"
```
