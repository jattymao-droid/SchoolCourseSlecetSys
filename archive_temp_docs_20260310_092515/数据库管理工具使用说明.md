# PostgreSQL数据库管理工具使用说明

## 概述

`manage-database-server.sh` 是一个功能强大的交互式PostgreSQL数据库管理工具，提供了完整的数据库和表管理功能。

## 主要功能

### 1. 数据库信息管理
- 查看数据库基本信息（大小、表数量、连接数等）
- 查看数据库版本
- 查看所有表列表及大小
- 查看数据库连接状态
- 查看表空间使用情况

### 2. 表管理功能
- 查看表结构（字段、类型、约束等）
- 查看表统计信息（记录数、大小、逻辑删除记录等）
- 查看表数据（支持自定义行数）
- 清空表数据（TRUNCATE）
- 删除表（DROP）
- 备份单个表

### 3. 数据清理功能
- 自动检测所有包含 `del_flag` 字段的表
- 批量清除逻辑删除记录（del_flag='2'）
- 统计每个表的逻辑删除记录数

### 4. 高级功能
- 执行自定义SQL语句
- 备份整个数据库
- 数据库优化（VACUUM ANALYZE）
- 交互式SQL控制台

## 安装和使用

### 步骤 1: 上传脚本到服务器

```bash
# 使用 scp 上传
scp manage-database-server.sh root@your-server:/root/

# 或使用宝塔面板的文件管理功能上传
```

### 步骤 2: 添加执行权限

```bash
chmod +x manage-database-server.sh
```

### 步骤 3: 运行脚本

```bash
./manage-database-server.sh
```

### 功能 1: 查看数据库信息

显示数据库的基本信息：
- 数据库名称和用户
- 数据库大小
- 表数量
- 当前连接数
- PostgreSQL版本

**示例输出**:
```
数据库名称: school-course
数据库用户: postgres

数据库大小: 45 MB
表数量: 28
当前连接数: 3

数据库版本信息:
PostgreSQL 14.5 on x86_64-pc-linux-gnu
```

### 功能 2: 列出所有表

显示数据库中所有表的列表，包括：
- Schema名称
- 表名
- 表大小

**示例输出**:
```
 schema |        table_name        |  size   
--------+--------------------------+---------
 public | stu_student_info         | 128 kB
 public | sys_user                 | 256 kB
 public | sys_course               | 64 kB
 public | stu_course_selection     | 192 kB
```

### 功能 3: 管理指定表

选择一个表后，进入表管理子菜单：

```
==========================================
表管理: stu_student_info
==========================================

1) 查看表结构
2) 查看表统计
3) 查看表数据 (前10行)
4) 查看表数据 (前50行)
5) 查看表数据 (自定义行数)
6) 清空表数据
7) 删除表
8) 备份表
9) 返回上级菜单

请选择操作 (1-9):
```

#### 3.1 查看表结构

显示表的完整结构定义：
- 字段名称和类型
- 约束条件
- 索引信息
- 外键关系

**示例输出**:
```
                Table "public.stu_student_info"
   Column    |          Type          | Nullable | Default 
-------------+------------------------+----------+---------
 id          | bigint                 | not null | 
 user_id     | bigint                 |          | 
 student_no  | character varying(50)  | not null | 
 real_name   | character varying(100) | not null | 
 class_id    | bigint                 |          | 
 grade_id    | bigint                 |          | 
 del_flag    | character(1)           |          | '0'
 create_time | timestamp              |          | 

Indexes:
    "stu_student_info_pkey" PRIMARY KEY, btree (id)
    "idx_student_no" UNIQUE, btree (student_no)
```

#### 3.2 查看表统计

显示表的统计信息：
- 总记录数
- 表大小
- 逻辑删除记录数（如果有del_flag字段）
- 活跃记录数

**示例输出**:
```
总记录数: 580
表大小: 128 kB
逻辑删除记录数: 290
活跃记录数: 290
```

#### 3.3 查看表数据

显示表中的数据记录，支持：
- 前10行（快速预览）
- 前50行（详细查看）
- 自定义行数

#### 3.4 清空表数据

使用 `TRUNCATE TABLE` 清空表中所有数据。

**注意**: 
- 此操作会删除所有记录
- 会重置自增序列
- 需要确认操作

#### 3.5 删除表

使用 `DROP TABLE` 删除整个表。

**警告**: 
- 此操作不可恢复
- 会删除表结构和所有数据
- 需要确认操作

#### 3.6 备份表

将单个表的数据和结构导出到SQL文件。

**输出文件**: `backup_表名_时间戳.sql`

### 功能 4: 清除逻辑删除记录

自动检测所有包含 `del_flag` 字段的表，并清除其中 `del_flag='2'` 的记录。

**工作流程**:
1. 扫描所有表，查找包含 `del_flag` 字段的表
2. 统计每个表的逻辑删除记录数
3. 显示汇总信息
4. 请求确认
5. 批量清除所有逻辑删除记录

**示例输出**:
```
找到以下包含 del_flag 字段的表:
stu_student_info
sys_user
sys_course

  stu_student_info: 290 条逻辑删除记录
  sys_user: 290 条逻辑删除记录
  sys_course: 0 条逻辑删除记录

总计: 580 条逻辑删除记录

确认清除所有逻辑删除记录吗？(yes/no):
```

### 功能 5: 执行自定义SQL

提供交互式SQL控制台，可以执行任意SQL语句。

**使用方法**:
```
SQL> SELECT COUNT(*) FROM stu_student_info;
 count 
-------
   290
(1 row)

SQL> SELECT student_no, real_name FROM stu_student_info LIMIT 5;
 student_no | real_name 
------------+-----------
 3101       | 张三
 3102       | 李四
 3103       | 王五

SQL> exit
```

**支持的SQL类型**:
- SELECT 查询
- INSERT 插入
- UPDATE 更新
- DELETE 删除
- CREATE/ALTER/DROP DDL语句
- 任何有效的PostgreSQL SQL语句

### 功能 6: 备份整个数据库

将整个数据库导出到SQL文件。

**输出文件**: `backup_数据库名_时间戳.sql`

**包含内容**:
- 所有表结构
- 所有表数据
- 索引和约束
- 序列和触发器

### 功能 7: 查看数据库连接

显示当前所有连接到数据库的会话信息：
- 进程ID (PID)
- 用户名
- 应用程序名称
- 客户端地址
- 连接状态
- 查询开始时间

**示例输出**:
```
 pid  | usename  | application_name | client_addr |  state  |         query_start         
------+----------+------------------+-------------+---------+-----------------------------
 1234 | postgres | psql             | 172.17.0.1  | active  | 2026-03-09 20:15:32.123456
 1235 | postgres | Java Application | 172.17.0.1  | idle    | 2026-03-09 20:10:15.654321
```

### 功能 8: 查看表空间使用情况

显示每个表的空间使用详情：
- 表名
- 总大小（表+索引）
- 表大小
- 索引大小

按总大小降序排列，方便识别占用空间最大的表。

**示例输出**:
```
 schemaname |      tablename       | total_size | table_size | index_size 
------------+----------------------+------------+------------+------------
 public     | sys_user             | 512 kB     | 256 kB     | 256 kB
 public     | stu_course_selection | 384 kB     | 192 kB     | 192 kB
 public     | stu_student_info     | 256 kB     | 128 kB     | 128 kB
```

### 功能 9: 优化数据库 (VACUUM)

执行 `VACUUM ANALYZE` 命令优化数据库：
- 回收已删除记录占用的空间
- 更新统计信息
- 提高查询性能

**建议使用场景**:
- 大量删除数据后
- 数据库性能下降时
- 定期维护

所有危险操作（清空表、删除表、清除数据）都需要输入 `yes` 确认。

### 4. 交互式菜单

清晰的菜单结构，易于导航和使用。

### 5. 智能统计

自动识别包含 `del_flag` 字段的表，提供逻辑删除记录统计。

## 常见问题

### Q1: 提示"未找到运行中的PostgreSQL容器"

**解决方法**:
```bash
# 检查容器状态
docker ps | grep postgres

# 启动容器
docker start <container_name>
```

### Q2: 提示"数据库连接失败"

**检查项**:
1. 数据库名称是否正确
2. 用户名和密码是否正确
3. 容器内PostgreSQL服务是否运行

**解决方法**:
```bash
# 进入容器检查
docker exec -it <container_name> psql -U postgres -l
```

### Q3: 备份文件保存在哪里？

备份文件保存在脚本运行的当前目录。

**查看备份文件**:
```bash
ls -lh backup_*.sql
```

### Q4: 如何恢复备份？

```bash
# 恢复整个数据库
docker exec -i <container_name> psql -U postgres -d school-course < backup_school-course_20260309_201234.sql

# 恢复单个表
docker exec -i <container_name> psql -U postgres -d school-course < backup_stu_student_info_20260309_201234.sql
```

### Q5: VACUUM优化需要多长时间？

取决于数据库大小和删除的数据量：
- 小型数据库（< 100MB）：几秒钟
- 中型数据库（100MB - 1GB）：几分钟
- 大型数据库（> 1GB）：可能需要更长时间

建议在业务低峰期执行。

## 安全建议

1. **备份优先**: 在执行任何删除或清空操作前，先备份数据
2. **谨慎删除**: 删除表操作不可恢复，务必确认
3. **权限控制**: 脚本具有完整数据库权限，注意使用权限
4. **定期备份**: 建议定期使用功能6备份整个数据库
5. **测试环境**: 在生产环境使用前，先在测试环境验证

## 高级用法

可以通过管道输入命令实现批量操作：

```bash
# 自动清除逻辑删除记录
echo -e "4\nyes\n10" | ./manage-database-server.sh
```

### 定时任务

配合cron实现定时维护：

```bash
# 每天凌晨2点优化数据库
0 2 * * * cd /root && echo -e "9\n10" | ./manage-database-server.sh
```

### 远程执行

通过SSH远程执行：

```bash

```
