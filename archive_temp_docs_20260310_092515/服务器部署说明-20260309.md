## 故障排查

### 问题1：后端启动失败

**检查日志：**
```bash
tail -f /www/wwwroot/course-selection/logs/backend.log
```

**常见原因：**
- 端口 8080 被占用：`netstat -tlnp | grep 8080`
- 数据库连接失败：检查 PostgreSQL 容器是否运行
- Redis 连接失败：检查 Redis 容器是否运行

**解决方法：**
```bash
# 检查 Docker 容器
docker ps | grep postgres
docker ps | grep redis

# 如果容器未运行，启动容器
docker start CONTAINER_NAME
```

### 问题2：前端访问 404

**检查 Nginx 配置：**
```bash
cat /www/server/panel/vhost/nginx/xk.xmls.vip.conf
```

**确认前端目录：**
```bash
ls -la /www/wwwroot/course-selection/frontend/
```

**重载 Nginx：**
```bash
nginx -t
nginx -s reload
```

### 问题3：数据库导入失败

**手动导入：**
```bash
# 查找 PostgreSQL 容器名
PG_CONTAINER=$(docker ps --format '{{.Names}}' | grep -i postgres | head -n 1)

# 删除旧数据库
docker exec $PG_CONTAINER psql -U postgres -c "DROP DATABASE \"school-course\";"

# 创建新数据库
docker exec $PG_CONTAINER psql -U postgres -c "CREATE DATABASE \"school-course\" OWNER postgres;"

# 导入数据
cd /root/deploy-server-20260309_200842
docker exec -i $PG_CONTAINER pg_restore -U postgres -d school-course -v < database_backup.dump
```

### 问题4：学生删除后重新导入失败

这个问题已经在本次更新中修复。如果仍然遇到：

1. 检查后端是否使用最新的 JAR 文件：
```bash
ls -lh /www/wwwroot/course-selection/backend/ruoyi-admin.jar
```

2. 检查数据库中是否有逻辑删除的记录：
```bash
PG_CONTAINER=$(docker ps --format '{{.Names}}' | grep -i postgres | head -n 1)
docker exec $PG_CONTAINER psql -U postgres -d school-course -c "SELECT COUNT(*) FROM stu_student_info WHERE del_flag = '2';"
```

3. 如果有逻辑删除的记录，清理它们：
```bash
docker exec $PG_CONTAINER psql -U postgres -d school-course -c "DELETE FROM stu_student_info WHERE del_flag = '2'; DELETE FROM sys_user WHERE user_type = '01' AND del_flag = '2';"
```
