## 故障排查

### 1. 后端启动失败

查看日志:
```bash
tail -f /www/wwwroot/course-selection/logs/backend.log
```

常见原因:
- PostgreSQL 容器未启动: `docker ps | grep postgres`
- Redis 容器未启动: `docker ps | grep redis`
- 端口 8080 被占用: `netstat -tunlp | grep 8080`
- 数据库连接失败: 检查数据库密码和容器名

### 2. 前端无法访问

检查:
- Nginx 是否运行: `systemctl status nginx`
- Nginx 配置是否正确: `nginx -t`
- 后端是否启动: `ps aux | grep ruoyi-admin.jar`

### 3. 数据库导入失败

手动导入:
```bash
# 查找 PostgreSQL 容器名
docker ps | grep postgres

# 删除旧数据库
docker exec <容器名> psql -U postgres -d postgres -c "DROP DATABASE IF EXISTS \"school-course\";"

# 创建新数据库
docker exec <容器名> psql -U postgres -d postgres -c "CREATE DATABASE \"school-course\" WITH ENCODING='UTF8';"

# 导入数据
docker exec -i <容器名> pg_restore -U postgres -d school-course -v < /www/wwwroot/course-selection/database/database_backup.dump
```

### 4. Nginx 配置错误

重新加载配置:
```bash
nginx -t  # 测试配置
nginx -s reload  # 重载配置
```
