# 学校选课系统 - 生产环境部署指南

## 一、本地打包步骤

### 1. 执行打包脚本

在Windows本地环境执行：

```powershell
cd SchoolCourseSlecetSys
.\build-production-package.ps1
```

脚本会自动完成：
- ✓ 构建后端Spring Boot应用（生产环境配置）
- ✓ 构建前端Vue应用
- ✓ 导出数据库结构和数据
- ✓ 生成部署脚本和配置文件
- ✓ 打包到 `deploy` 目录

### 2. 打包完成后的目录结构

```
deploy/
├── backend/              # 后端JAR包
│   └── ruoyi-admin.jar
├── frontend/             # 前端静态文件
│   ├── index.html
│   └── static/
├── scripts/              # 部署脚本
│   ├── deploy.sh        # 一键部署脚本
│   ├── start-backend.sh # 启动后端
│   ├── stop-backend.sh  # 停止后端
│   └── nginx.conf       # Nginx配置
├── sql/                  # 数据库文件
│   ├── schema.sql       # 数据库结构
│   ├── data.sql         # 数据库数据
│   └── full_backup.sql  # 完整备份
└── README.md            # 部署说明
```

## 三、服务器部署步骤

### 方式一：一键部署（推荐）

#### 1. 上传部署包到服务器

```bash
# 在本地执行（将deploy目录上传到服务器）
scp -r deploy root@your-server-ip:/opt/

# 或使用宝塔面板的文件管理功能上传
```

#### 2. 执行一键部署脚本

```bash
# SSH登录到服务器
ssh root@your-server-ip

# 进入部署目录
cd /opt/deploy/scripts

# 添加执行权限
chmod +x deploy.sh

# 执行部署
./deploy.sh
```

脚本会自动完成：
1. ✓ 检查服务器环境（Java、Docker、PostgreSQL、Redis）
2. ✓ 创建部署目录 `/opt/school-course`
3. ✓ 复制所有文件到部署目录
4. ✓ 创建数据库 `school-course`
5. ✓ 导入数据库结构和数据
6. ✓ 启动后端服务（端口8080）
7. ✓ 生成Nginx配置文件

#### 3. 配置Nginx（使用宝塔面板）

在宝塔面板中：

1. **创建网站**
   - 点击"网站" → "添加站点"
   - 域名：填写你的域名或服务器IP
   - 根目录：`/opt/school-course/frontend`
   - PHP版本：纯静态

2. **配置反向代理**
   - 点击网站设置 → "反向代理"
   - 添加反向代理：
     - 代理名称：`backend-api`
     - 目标URL：`http://127.0.0.1:8080`
     - 发送域名：`$host`
     - 内容替换：留空
   - 配置文件位置：`/opt/school-course/nginx.conf`（可参考）

3. **保存并重启Nginx**

#### 4. 访问系统

- 前端地址：`http://your-server-ip`
- 默认账号：`admin`
- 默认密码：`admin123`

### 方式二：手动部署

如果一键部署脚本遇到问题，可以按以下步骤手动部署：

#### 1. 创建部署目录

```bash
mkdir -p /opt/school-course/{backend,frontend,sql}
```

#### 2. 上传文件

将 `deploy` 目录中的文件分别上传到对应目录：
- `backend/ruoyi-admin.jar` → `/opt/school-course/backend/`
- `frontend/*` → `/opt/school-course/frontend/`
- `sql/*` → `/opt/school-course/sql/`

#### 3. 初始化数据库

```bash
cd /opt/school-course/sql

# 设置PostgreSQL密码
export PGPASSWORD=mm5621528

# 获取PostgreSQL容器名称
POSTGRES_CONTAINER=$(docker ps | grep postgres | awk '{print $NF}' | head -n 1)

# 创建数据库
docker exec -e PGPASSWORD=$PGPASSWORD $POSTGRES_CONTAINER psql -U postgres -c "CREATE DATABASE \"school-course\";"

# 导入数据库
docker exec -i -e PGPASSWORD=$PGPASSWORD $POSTGRES_CONTAINER psql -U postgres -d school-course < full_backup.sql
```

#### 4. 启动后端服务

```bash
cd /opt/school-course/backend

# 后台启动
nohup java -jar ruoyi-admin.jar --spring.profiles.active=prod > backend.log 2>&1 &

# 查看日志
tail -f backend.log
```

#### 5. 配置Nginx

参考"方式一"中的Nginx配置步骤。

## 五、故障排查

### 1. 后端无法启动

**检查Java版本：**
```bash
java -version
```

**检查端口占用：**
```bash
netstat -tlnp | grep 8080
```

**查看详细日志：**
```bash
tail -f /opt/school-course/backend/backend.log
```

### 2. 数据库连接失败

**检查PostgreSQL容器状态：**
```bash
docker ps | grep postgres
```

**测试数据库连接：**
```bash
docker exec -it $(docker ps | grep postgres | awk '{print $1}') psql -U postgres -d school-course
```

**检查数据库配置：**
- 数据库名称：`school-course`
- 用户名：`postgres`
- 密码：`mm5621528`
- 端口：`5432`

### 3. Redis连接失败

**检查Redis容器状态：**
```bash
docker ps | grep redis
```

**测试Redis连接：**
```bash
docker exec -it $(docker ps | grep redis | awk '{print $1}') redis-cli ping
```

### 4. 前端无法访问

**检查Nginx状态：**
```bash
systemctl status nginx
```

**检查Nginx配置：**
```bash
nginx -t
```

**重启Nginx：**
```bash
systemctl restart nginx
```

### 5. 后端API无法访问

**检查后端服务状态：**
```bash
ps aux | grep ruoyi-admin.jar
```

**测试后端API：**
```bash
curl http://localhost:8080
```

**检查防火墙：**
```bash
firewall-cmd --list-ports
firewall-cmd --add-port=8080/tcp --permanent
firewall-cmd --reload
```

## 六、数据库配置说明

### 生产环境配置文件

配置文件位置：`ruoyi-admin/src/main/resources/application-druid-prod.yml`

```yaml
spring:
  datasource:
    druid:
      master:
        url: jdbc:postgresql://localhost:5432/school-course
        username: postgres
        password: mm5621528
```

### 数据库连接池配置

- 初始连接数：5
- 最小空闲连接：10
- 最大活动连接：20
- 连接超时：30秒
- 查询超时：60秒

## 七、性能优化建议

### 1. JVM参数优化

编辑启动脚本，添加JVM参数：

```bash
java -Xms512m -Xmx1024m -XX:+UseG1GC \
     -jar ruoyi-admin.jar --spring.profiles.active=prod
```

### 2. 数据库连接池优化

根据服务器配置调整连接池参数：
- 小型服务器（2核4G）：maxActive=20
- 中型服务器（4核8G）：maxActive=50
- 大型服务器（8核16G）：maxActive=100

### 3. Nginx缓存优化

在Nginx配置中启用缓存：

```nginx
# 静态资源缓存
location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg)$ {
    expires 30d;
    add_header Cache-Control "public, immutable";
}
```
