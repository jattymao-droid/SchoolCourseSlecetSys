## 🐛 常见问题

### Q1: 导入时提示"数据库不存在"

**解决**：先创建数据库
```bash
docker exec postgresql_d6wx-postgresql_D6Wx-1 psql -U postgres -c "CREATE DATABASE \"school-course\" WITH ENCODING='UTF8';"
```

### Q2: 导入时提示"权限不足"

**解决**：确保使用 postgres 超级用户
```bash
docker exec postgresql_d6wx-postgresql_D6Wx-1 psql -U postgres -d school-course < /path/to/complete-database.sql
```

### Q3: 导入后表是空的

**原因**：可能使用了错误的 SQL 文件

**解决**：确保使用 `complete-database.sql`（717 KB），而不是其他备份文件。

### Q4: 后端仍然报错"relation does not exist"

**检查**：
```bash
# 1. 确认数据库名称正确
docker exec postgresql_d6wx-postgresql_D6Wx-1 psql -U postgres -l | grep school-course

# 2. 确认表已导入
docker exec postgresql_d6wx-postgresql_D6Wx-1 psql -U postgres -d school-course -c "\dt"

# 3. 检查后端配置中的数据库名称
# 应该是：jdbc:postgresql://localhost:5432/school-course
```

### Q5: 导入过程中断或超时

**解决**：
1. 检查 SQL 文件是否完整（717 KB）
2. 增加超时时间或分批导入
3. 检查磁盘空间是否充足
