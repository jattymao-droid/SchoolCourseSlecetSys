## 🚀 部署步骤（3步完成）

### 第1步：上传部署包

**方式A - 使用SCP命令**
```bash
scp -r deploy root@your-server-ip:/opt/
```

**方式B - 使用宝塔面板**
1. 登录宝塔面板
2. 文件管理 → 上传
3. 将 `deploy` 目录上传到 `/opt/`

### 第2步：执行一键部署

SSH登录服务器后执行：
```bash
cd /opt/deploy/scripts
chmod +x deploy.sh
./deploy.sh
```

**脚本会自动完成：**
- ✅ 检查环境（Java、Docker、PostgreSQL、Redis）
- ✅ 创建部署目录 `/opt/school-course`
- ✅ 复制所有文件
- ✅ 创建数据库 `school-course`
- ✅ 导入数据库数据
- ✅ 启动后端服务（端口8080）
- ✅ 生成服务管理脚本

### 第3步：配置Nginx

在宝塔面板中：

1. **创建网站**
   - 域名：你的域名或服务器IP
   - 根目录：`/opt/school-course/frontend`
   - PHP版本：纯静态

2. **配置反向代理**
   - 点击网站设置 → 反向代理 → 添加反向代理
   - 代理名称：`backend-api`
   - 目标URL：`http://127.0.0.1:8080`
   - 发送域名：`$host`

3. **保存并重启Nginx**

## 🔍 故障排查

### 问题1：后端无法启动

**检查步骤：**
```bash
# 1. 检查Java版本
java -version

# 2. 检查端口占用
netstat -tlnp | grep 8080

# 3. 查看详细日志
tail -f /opt/school-course/backend/backend.log
```

### 问题2：数据库连接失败

**检查步骤：**
```bash
# 1. 检查PostgreSQL容器状态
docker ps | grep postgres

# 2. 测试数据库连接
docker exec -it $(docker ps | grep postgres | awk '{print $1}') psql -U postgres -d school-course

# 3. 检查数据库密码是否正确（mm5621528）
```

### 问题3：前端无法访问

**检查步骤：**
```bash
# 1. 检查Nginx状态
systemctl status nginx

# 2. 检查Nginx配置
nginx -t

# 3. 重启Nginx
systemctl restart nginx
```

### 问题4：Redis连接失败

**检查步骤：**
```bash
# 1. 检查Redis容器状态
docker ps | grep redis

# 2. 测试Redis连接
docker exec -it $(docker ps | grep redis | awk '{print $1}') redis-cli ping
```
