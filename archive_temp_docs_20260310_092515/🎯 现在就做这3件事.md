## 第 1 件事：修复 Nginx 配置（2 分钟）

### 操作步骤

1. 打开宝塔面板
2. 网站 → xk.xmls.vip → 设置 → 配置文件
3. **全选删除**现有配置
4. 复制粘贴以下内容：

```nginx
server {
    listen 80;
    server_name xk.xmls.vip;
    index index.html;
    root /www/wwwroot/xk.xmls.vip/frontend;
    
    location /prod-api/ {
        proxy_pass http://127.0.0.1:8080/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
    
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    access_log /www/wwwlogs/xk.xmls.vip.log;
    error_log /www/wwwlogs/xk.xmls.vip.error.log;
}
```

5. 点击"保存"
6. SSH 执行：`nginx -s reload`

- 宝塔面板显示"保存成功"
- 没有语法错误提示

## ❌ 如果出现问题

### 问题 1：Nginx 配置仍然保存失败

**解决**：
- 确保使用纯文本复制（不要用 Word）
- 检查是否有多余的空格或特殊字符
- 尝试在宝塔面板中"重置配置"后再粘贴

### 问题 2：后端启动失败

**检查日志**：
```bash
cat /www/wwwroot/xk.xmls.vip/backend/backend.log
```

**常见原因**：
- 数据库连接失败 → 检查 PostgreSQL 是否运行
- Redis 连接失败 → 检查 Redis 是否运行
- 端口被占用 → 检查 8080 端口是否被占用

**解决方法**：
```bash
# 检查 PostgreSQL
docker ps | grep postgresql

# 检查 Redis
systemctl status redis

# 检查端口
netstat -tlnp | grep 8080
```

### 问题 3：访问网站显示 502

**原因**：后端未启动或端口不对

**解决**：
```bash
# 重启后端
cd /www/wwwroot/xk.xmls.vip/backend
./restart.sh

# 查看日志
tail -f backend.log
```
