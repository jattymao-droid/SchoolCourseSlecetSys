# 数据库问题快速解决指南

## 当前问题

你在使用 `manage-database-server.sh` 时发现 `school-course` 数据库是空的（0 rows），这说明数据库配置可能有问题。

## 快速解决步骤

### 第一步：运行诊断脚本

上传并运行诊断脚本，找出实际使用的数据库：

```bash
# 1. 上传诊断脚本到服务器
scp diagnose-database-server.sh root@your-server:/root/

# 2. 添加执行权限
chmod +x diagnose-database-server.sh

# 3. 运行诊断
./diagnose-database-server.sh
```

诊断脚本会告诉你：
- 所有数据库列表及大小
- 每个数据库的表数量
- 后端应用实际连接的数据库
- 自动给出修复建议

### 第二步：根据诊断结果选择解决方案

#### 情况 A: 数据在 `ry_vue` 数据库中

如果诊断显示 `ry_vue` 数据库有数据，而 `school-course` 是空的：

**解决方法：修改管理脚本连接到 `ry_vue`**

```bash
# 方法1: 使用 sed 命令修改
sed -i 's/DB_NAME="school-course"/DB_NAME="ry_vue"/' manage-database-server.sh
sed -i 's/DB_NAME="school-course"/DB_NAME="ry_vue"/' clean-database-server.sh
sed -i 's/DB_NAME="school-course"/DB_NAME="ry_vue"/' clean-deleted-records-server.sh

# 方法2: 手动编辑文件
vi manage-database-server.sh
# 找到第18行: DB_NAME="school-course"
# 改为: DB_NAME="ry_vue"
```

然后重新运行管理脚本：

```bash
./manage-database-server.sh
```

#### 情况 B: `school-course` 数据库应该有数据但是空的

如果 `school-course` 是正确的数据库名称，但数据没有导入成功：

**解决方法：重新导入数据库**

```bash
# 1. 进入部署目录
cd /root/deploy-server-20260309_200842

# 2. 检查备份文件是否存在
ls -lh database_backup.dump

# 3. 重新导入数据库
docker exec -i postgresql_npxc-postgresql_NpXc-1 pg_restore -U postgres -d school-course -v < database_backup.dump

# 4. 验证导入结果
docker exec -i postgresql_npxc-postgresql_NpXc-1 psql -U postgres -d school-course -c "\dt"
```

#### 情况 C: 需要从 `ry_vue` 复制数据到 `school-course`

如果你想保持使用 `school-course` 数据库，但数据在 `ry_vue` 中：

**解决方法：复制数据库**

```bash
# 1. 导出 ry_vue 数据库
docker exec -i postgresql_npxc-postgresql_NpXc-1 pg_dump -U postgres ry_vue > ry_vue_backup.sql

# 2. 删除并重建 school-course 数据库
docker exec -i postgresql_npxc-postgresql_NpXc-1 psql -U postgres -c "DROP DATABASE IF EXISTS \"school-course\";"
docker exec -i postgresql_npxc-postgresql_NpXc-1 psql -U postgres -c "CREATE DATABASE \"school-course\" OWNER postgres;"

# 3. 导入数据到 school-course
docker exec -i postgresql_npxc-postgresql_NpXc-1 psql -U postgres -d school-course < ry_vue_backup.sql

# 4. 重启后端服务
/www/wwwroot/course-selection/backend/restart.sh
```

### 第三步：验证修复结果

运行管理脚本验证：

```bash
./manage-database-server.sh
# 选择 2 - 列出所有表
# 应该能看到所有业务表
```

## 常用命令速查

### 查看所有数据库

```bash
docker exec -it postgresql_npxc-postgresql_NpXc-1 psql -U postgres -c "\l"
```

### 查看指定数据库的表

```bash
# 查看 school-course
docker exec -it postgresql_npxc-postgresql_NpXc-1 psql -U postgres -d school-course -c "\dt"

# 查看 ry_vue
docker exec -it postgresql_npxc-postgresql_NpXc-1 psql -U postgres -d ry_vue -c "\dt"
```

### 查看数据库大小

```bash
docker exec -it postgresql_npxc-postgresql_NpXc-1 psql -U postgres -c "
SELECT 
    datname as database_name,
    pg_size_pretty(pg_database_size(datname)) as size
FROM pg_database 
WHERE datistemplate = false
ORDER BY pg_database_size(datname) DESC;
"
```

### 查看后端连接的数据库

```bash
docker exec -it postgresql_npxc-postgresql_NpXc-1 psql -U postgres -c "
SELECT datname, usename, application_name, client_addr 
FROM pg_stat_activity 
WHERE datname IS NOT NULL;
"
```

### 查看后端日志

```bash
tail -f /www/wwwroot/course-selection/logs/backend.log | grep -i "database\|jdbc"
```

## 推荐的最佳实践

### 1. 统一数据库命名

建议在所有环境使用统一的数据库名称：
- 开发环境：`ry_vue`
- 生产环境：`school-course`

### 2. 配置文件管理

确保配置文件正确：
- 开发环境：`application-druid.yml` → `ry_vue`
- 生产环境：`application-druid-prod.yml` → `school-course`

### 3. 部署前检查清单

每次部署前检查：
- [ ] 数据库备份文件存在
- [ ] 数据库名称配置正确
- [ ] 后端配置文件正确
- [ ] 数据库连接密码正确

### 4. 定期备份

设置定时备份任务：

```bash
# 创建备份脚本
cat > /root/backup-database.sh << 'EOF'
#!/bin/bash
BACKUP_DIR="/root/database-backups"
DATE=$(date +%Y%m%d_%H%M%S)
mkdir -p $BACKUP_DIR

# 备份数据库
docker exec -i postgresql_npxc-postgresql_NpXc-1 pg_dump -U postgres school-course > $BACKUP_DIR/school-course_$DATE.sql

# 保留最近7天的备份
find $BACKUP_DIR -name "*.sql" -mtime +7 -delete

echo "备份完成: $BACKUP_DIR/school-course_$DATE.sql"
EOF

chmod +x /root/backup-database.sh

# 添加到 crontab（每天凌晨2点备份）
(crontab -l 2>/dev/null; echo "0 2 * * * /root/backup-database.sh") | crontab -
```
