# 修复管理员账号问题

## 问题原因

前后端都启动成功，但是登录时报错"用户名或密码错误"。

经过检查发现：
- ✅ 数据库连接正常
- ✅ 后端服务运行正常
- ❌ **数据库中缺少 admin 管理员账号**

数据库备份文件中只有 `user_id=2` 的 'ry' 用户，没有 `user_id=1` 的 'admin' 用户。

---

## 解决方案

### 方案 1：执行修复 SQL 脚本（推荐）

#### 步骤 1：上传修复脚本

将本地文件上传到服务器：
- **本地文件**：`SchoolCourseSlecetSys/deploy/sql/fix-admin-user.sql`
- **服务器路径**：`/www/wwwroot/xk.xmls.vip/sql/fix-admin-user.sql`

#### 步骤 2：执行修复脚本

```bash

docker exec postgresql_d6wx-postgresql_D6Wx-1 psql -U postgres -d school-course -f /tmp/fix-admin-user.sql
```

或者直接在服务器上执行：

```bash

docker exec -i postgresql_d6wx-postgresql_D6Wx-1 psql -U postgres -d school-course << 'EOF'

    create_by, create_time, update_by, update_time, remark
) VALUES (
    1, 103, 'admin', '管理员', '00',
    'admin@ruoyi.com', '15888888888', '1', '', 
    '$2a$10$7JB720yubVSZvUI0rEqK/.VqGOZTH.ulu33dHOiBE8ByOhJIrdAu2',
    '0', '0', '127.0.0.1', NOW(), NOW(),
    'admin', NOW(), '', NULL, '管理员'
);

-- 分配超级管理员角色
DELETE FROM public.sys_user_role WHERE user_id = 1;
INSERT INTO public.sys_user_role (user_id, role_id) VALUES (1, 1);

-- 重置序列

-- 验证结果
SELECT user_id, user_name, nick_name, status FROM public.sys_user WHERE user_name = 'admin';
EOF
```

#### 步骤 3：验证修复结果

## 常见问题

### Q1: 执行 SQL 后仍然无法登录

**可能原因**：
1. 密码加密方式不对
2. 角色权限未分配
3. 用户状态不正常

**解决方法**：
```bash
# 检查用户状态
docker exec postgresql_d6wx-postgresql_D6Wx-1 psql -U postgres -d school-course -c "SELECT user_id, user_name, status, del_flag FROM sys_user WHERE user_name = 'admin';"

# status 应该是 '0'（正常）
# del_flag 应该是 '0'（未删除）
```

### Q2: 提示"该账号已被停用"

**解决方法**：
```bash
# 启用账号
docker exec postgresql_d6wx-postgresql_D6Wx-1 psql -U postgres -d school-course -c "UPDATE sys_user SET status = '0' WHERE user_name = 'admin';"
```

### Q3: 提示"该账号已被删除"

**解决方法**：
```bash
# 恢复账号
docker exec postgresql_d6wx-postgresql_D6Wx-1 psql -U postgres -d school-course -c "UPDATE sys_user SET del_flag = '0' WHERE user_name = 'admin';"
```

### Q4: 登录后没有任何菜单权限

**解决方法**：
```bash
# 检查角色分配
docker exec postgresql_d6wx-postgresql_D6Wx-1 psql -U postgres -d school-course -c "SELECT * FROM sys_user_role WHERE user_id = 1;"

# 如果没有记录，执行：
docker exec postgresql_d6wx-postgresql_D6Wx-1 psql -U postgres -d school-course -c "INSERT INTO sys_user_role (user_id, role_id) VALUES (1, 1);"
```

## 技术说明

### 密码加密

系统使用 BCrypt 算法加密密码：
- **明文密码**：`admin123`
- **加密后**：`$2a$10$7JB720yubVSZvUI0rEqK/.VqGOZTH.ulu33dHOiBE8ByOhJIrdAu2`

这是 RuoYi 框架的默认密码加密方式。

### 用户状态

- `status = '0'`：正常
- `status = '1'`：停用

### 删除标志

- `del_flag = '0'`：未删除
- `del_flag = '2'`：已删除

### 用户类型

- `user_type = '00'`：系统用户（管理员）
- `user_type = '01'`：学生
- `user_type = '02'`：教师
