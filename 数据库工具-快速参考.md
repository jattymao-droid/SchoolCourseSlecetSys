## 📋 工具功能速查

### diagnose-database-server.sh 🔍
```
功能: 诊断数据库状态
输出: 
  - 所有数据库列表
  - 表数量统计
  - 连接状态
  - 修复建议
```

### auto-fix-database-server.sh 🔧
```
功能: 自动修复配置问题
选项:
  1. 修改脚本连接到正确数据库
  2. 复制数据库
  3. 重新导入数据
```

### manage-database-server.sh 📊
```
功能: 完整数据库管理
菜单:
  1. 查看数据库信息
  2. 列出所有表
  3. 管理指定表
  4. 清除逻辑删除记录
  5. 执行自定义SQL
  6. 备份整个数据库
  7. 查看数据库连接
  8. 查看表空间使用
  9. 优化数据库
```

### clean-database-server.sh 🧹
```
功能: 清理数据
模式:
  1. 清除逻辑删除记录
  2. 清除所有学生数据
  3. 清除所有课程数据
  4. 清除所有选课数据
  5. 清除所有业务数据
```

### clean-deleted-records-server.sh 🗑️
```
功能: 清除逻辑删除记录
清理:
  - stu_student_info (del_flag='2')
  - sys_user (del_flag='2')
```

## 💻 常用命令

### 查看容器
```bash
docker ps | grep postgres
```

### 查看所有数据库
```bash
docker exec -it <container> psql -U postgres -c "\l"
```

### 查看表列表
```bash
docker exec -it <container> psql -U postgres -d school-course -c "\dt"
```

### 查看表数据
```bash
docker exec -it <container> psql -U postgres -d school-course -c "SELECT * FROM stu_student_info LIMIT 10;"
```

### 手动清除逻辑删除
```bash
docker exec -it <container> psql -U postgres -d school-course -c "DELETE FROM stu_student_info WHERE del_flag='2';"
```

### 备份数据库
```bash
docker exec -i <container> pg_dump -U postgres school-course > backup.sql
```

### 恢复数据库
```bash
docker exec -i <container> psql -U postgres -d school-course < backup.sql
```

## 🆘 故障排查

### 问题: 未找到容器
```bash
# 检查容器
docker ps -a | grep postgres

# 启动容器
docker start <container_name>
```

### 问题: 连接失败
```bash
# 测试连接
docker exec -it <container> psql -U postgres -c "SELECT 1;"

# 检查密码
# 编辑脚本修改 DB_PASSWORD
```

### 问题: 数据库是空的
```bash
# 运行自动修复
./auto-fix-database-server.sh

# 或手动导入
docker exec -i <container> pg_restore -U postgres -d school-course < backup.dump
```
